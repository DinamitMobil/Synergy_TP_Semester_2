## 1. Обзор WEB-ориентированных информационных систем

Ниже приведён краткий анализ нескольких распространённых систем, работающих по архитектуре WEB-приложений.

### 1.1 1С:Предприятие (WEB-клиент)

**Назначение:** учёт, ERP, бухгалтерия, зарплата и кадры, документооборот.

**Архитектура:**

- Трёхзвенная схема: клиент (браузер) → сервер 1С → СУБД (чаще MS SQL Server или PostgreSQL).
- WEB-клиент реализован через HTTP(S), основная бизнес-логика выполняется на сервере 1С.
- Доступ возможен как из локальной сети, так и через интернет (при правильной настройке публикации).

**Возможности использования в компании:**

- Централизованный бухгалтерский учёт и расчёт зарплаты.
- Общая база справочников (клиенты, номенклатура, договора).
- Доступ пользователей из филиалов через браузер без установки толстого клиента.

---

### 1.2 Битрикс24 (облачный и коробочный)

**Назначение:** CRM, задачи и проекты, телефония, корпоративный портал.

**Архитектура:**

- WEB-приложение, работающее по модели клиент–сервер.
- В облачной версии серверная часть расположена в дата-центрах Bitrix, доступ к системе идёт через браузер.
- Используется PHP-приложение на веб-сервере (обычно nginx + Apache) и база данных (MySQL/MariaDB).

**Возможности использования в компании:**

- Ведение CRM (сделки, лиды, контакты).
- Постановка задач и контроль исполнения.
- Интеграция с телефонией и почтой.
- Быстрый старт без развёртывания собственной инфраструктуры (облачная модель).

---

### 1.3 SAP Fiori

**Назначение:** набор WEB-приложений для работы с SAP (ERP, SCM, HR и др.).

**Архитектура:**

- Фронтенд: web-клиент на основе SAPUI5 (JavaScript, HTML5, CSS).
- Сервер приложений: SAP Gateway / SAP NetWeaver, доступ по протоколу OData/HTTP(S).
- Бизнес-логика и данные — в ядре SAP ERP/S4HANA.

**Возможности использования в компании:**

- Удобный WEB-интерфейс к сложным ERP-процессам (согласование заявок, командировки, отпуск, закупки).
- Доступ к корпоративным процессам с мобильных устройств без установки толстого клиента.
- Унификация интерфейсов для разных ролей (руководитель, менеджер, бухгалтер и т.п.).

---

### 1.4 Odoo

**Назначение:** модульная ERP-система с WEB-интерфейсом (CRM, продажи, склад, производство и т.д.).

**Архитектура:**

- Сервер приложений на Python (Odoo Server).
- WEB-клиент в браузере (JavaScript + HTML).
- База данных PostgreSQL.
- Модульная структура (ядро + набор приложений).

**Возможности использования в компании:**

- Быстрый запуск CRM, управления продажами и складом в одной системе.
- Расширение функциональности за счёт установки дополнительных модулей.
- Возможность доработки модулей под свои бизнес-процессы (открытый исходный код).

---

### 1.5 Jira / Confluence (Atlassian)

**Назначение:** управление задачами и проектами (Jira), корпоративная база знаний и документация (Confluence).

**Архитектура:**

- WEB-приложения, доступные через браузер.
- Серверная часть на Java, работает через встроенный или внешний веб-сервер (Tomcat).
- База данных: PostgreSQL, MySQL/MariaDB, MS SQL Server или Oracle (зависит от конфигурации).

**Возможности использования в компании:**

- Управление задачами по Agile-подходам (Kanban, Scrum).
- Ведение технической документации, регламентов, инструкций.
- Интеграция с системами разработки (Git, CI/CD) и службой поддержки.

## 2. Общие выводы по архитектуре WEB-приложений

2.1 **Клиент–серверная модель.**  
   Во всех рассмотренных системах пользовательская часть работает в браузере, а бизнес-логика и доступ к данным находятся на стороне сервера.

2.2 **Использование СУБД.**  
   В качестве серверов баз данных чаще всего применяются **MS SQL Server, PostgreSQL, MySQL/MariaDB**, что обеспечивает надёжное хранение и масштабирование данных.

2.3 **Доступ из любой точки.**  
   WEB-архитектура позволяет пользователям работать с системой через интернет, что удобно для распределённых компаний и удалённой работы.

2.4 **Масштабируемость и расширяемость.**  
   Большинство систем поддерживают подключение модулей, плагинов и интеграцию через API (REST/OData и т.п.), что позволяет адаптировать продукт под конкретные бизнес-процессы.

2.5 **Подход к использованию в компании.**  
   Web-приложения целесообразно использовать для:
   - централизованного учёта и аналитики;
   - автоматизации документооборота и задач;
   - взаимодействия сотрудников и подразделений;
   - предоставления клиентам личных кабинетов и сервисов самообслуживания.

## 3. WEB-приложение (Delphi 10.2 + IIS + MS SQL Server)

**3.1 Стек:**

- Delphi 10.2 (WebBroker) — серверное приложение, обрабатывающее HTTP-запросы.
- MS Internet Information Server (IIS) — веб-сервер, через который пользователи обращаются к приложению.
- MS SQL Server — СУБД для хранения данных.

**3.2 Как запустить проект:**
- Выполнить скрипт sql/01_create_database.sql в SQL Server Management Studio.
- Открыть проект в Delphi 10.2.
- Собрать ISAPI DLL.
- В IIS создать приложение и указать путь к собранной DLL.
- Открыть приложение в браузере.
